## 各角色文档职责

| 角色 | 主要负责目录 | 协作目录 |
|------|-------------|----------|
| **产品经理** | `01-需求文档/` | `02-设计文档/`, `06-项目管理/` |
| **UI/UX设计师** | `02-设计文档/UI设计/`, `02-设计文档/交互设计/` | `01-需求文档/` |
| **技术经理/架构师** | `02-设计文档/架构设计/` | `03-开发文档/`, `05-运维文档/` |
| **后端开发** | `03-开发文档/后端开发/` | `02-设计文档/架构设计/`, `04-测试文档/` |
| **前端开发** | `03-开发文档/前端开发/` | `02-设计文档/UI设计/`, `04-测试文档/` |
| **测试工程师** | `04-测试文档/` | `01-需求文档/`, `03-开发文档/` |
| **运维工程师** | `05-运维文档/` | `03-开发文档/后端开发/部署文档.md` |
| **项目经理** | `06-项目管理/` | 所有目录 |

## 文档命名规范

1. **文件命名**: `类型-描述-版本.md`，如 `PRD-PDF转换功能-v1.0.md`
2. **版本号规则**: 主版本.次版本.修订号 (v1.0.0)
3. **日期格式**: YYYY-MM-DD

## 文档更新流程

1. 创建/修改文档 → 2. 提交到对应目录 → 3. 更新文档版本号 → 4. 通知相关人员 → 5. 归档旧版本
