# PDF工具集 UI设计规范（Web + 微信小程序）

## 1. 设计目标与范围

本规范基于 `doc/01-产品/` 下 PRD、用户故事与核心功能详设整理，聚焦：
- Web 端：专业、高效、可批量处理
- 微信小程序：轻量、快速、易分享
- MVP 优先：上传、处理、进度、下载的闭环体验

适用功能范围（优先级 P0/P1）：
- PDF 转换（PDF/Office 双向）
- PDF 合并、拆分、压缩
- OCR（基础入口与流程）
- 历史记录、会员引导、结果下载

---

## 2. 设计系统（来自 ui-ux-pro-max）

### 2.1 视觉基调
- 设计模式：简洁单列 + 功能卡片扩展
- 风格：专业、现代、轻量，强调清晰层级与高可读性
- 关键体验：降低首次使用门槛，3 步内进入上传操作

### 2.2 色彩 Tokens

| 语义 | 色值 | 用途 |
|------|------|------|
| `Primary` | `#0891B2` | 主品牌、关键高亮 |
| `Secondary` | `#22D3EE` | 信息辅助、标签背景 |
| `CTA` | `#22C55E` | 主按钮（开始处理/下载） |
| `Background` | `#ECFEFF` | 页面浅底、信息区分层 |
| `Text` | `#164E63` | 主文案与标题 |
| `Success` | `#16A34A` | 成功状态 |
| `Warning` | `#F59E0B` | 风险提示 |
| `Error` | `#DC2626` | 失败提示 |

使用约束：
- 主 CTA 与文字对比度不低于 4.5:1
- 大面积内容区优先白底，避免阅读疲劳
- 错误状态使用颜色 + 图标 + 文案三重表达

### 2.3 字体与排版
- 字体：`Plus Jakarta Sans`（中英统一，现代且清晰）
- 标题层级：
  - H1: 36/44, 700
  - H2: 28/36, 700
  - H3: 22/30, 600
- 正文字号：
  - Body-L: 16/26, 400
  - Body-M: 14/22, 400
  - Caption: 12/18, 400
- 数字信息（大小、进度、比例）建议使用半等宽数字样式

### 2.4 间距与圆角
- 8px 栅格体系：`4/8/12/16/24/32/48`
- 卡片内边距：`24px`（移动端 `16px`）
- 按钮高度：`44px`（移动端最小可点击区域）
- 圆角：容器 `16px`，输入与按钮 `10-12px`

### 2.5 动效
- Hover/Focus 过渡：`200-300ms`
- 禁止导致布局抖动的缩放 hover
- 进度动画平滑，避免“卡住无反馈”的感知
- 尊重 `prefers-reduced-motion`

## 3. 信息架构（Web）

### 3.1 顶层结构
1. 首页（价值主张 + 工具入口）
2. 工具工作台（统一上传与处理框架）
3. 结果页（下载、再处理、分享）
4. 历史记录（可选）
5. 会员与定价（转化）

### 3.2 首页区块顺序
1. Hero（核心价值 + 主 CTA）
2. 高频工具网格（6-8 项）
3. 热门场景（学生/行政/销售）
4. 信任背书（数据、评价、安全说明）
5. FAQ（格式、时效、隐私）
6. 小程序入口（扫码 + 场景说明）

## 4. 核心页面 UI 方案

## 4.1 工具工作台（统一模版）
- 页面结构：功能标题区 + 上传区 + 参数区 + 任务区 + 操作区
- 上传区支持：点击、拖拽、批量追加
- 任务区状态：等待/上传中/处理中/完成/失败
- 操作区固定主按钮：`开始处理`，并显示可用条件

关键组件：
- 文件行项：文件名、大小、状态、进度、删除、重试
- 参数面板：折叠式，默认展示高频选项
- 全局提示：顶部消息条 + 行内错误

## 4.2 PDF合并页（MVP）
- 强化“顺序即结果”的视觉提示
- 文件列表支持拖拽排序（附上移/下移备用操作）
- `开始合并` 仅在“就绪文件 >= 2 且无失败项”时可用
- 合并中显示阶段文案：
  - 上传中
  - 正在合并第 i/n 个文件
  - 正在生成下载文件

失败场景表现：
- 行级错误（文件格式/超限）
- 任务级错误（引擎失败/超时）
- 每个错误必须带“下一步操作”（重试、替换文件、分批处理）

## 4.3 结果页
- 主信息：文件名、大小、处理耗时、成功提示
- 主 CTA：`下载文件`
- 次 CTA：`再处理一组`、`返回工具列表`
- 可选：结果有效期说明（如 24h）

## 4.4 会员转化位
- 插入点：超限提示、批量上限触达、历史记录能力入口
- 呈现原则：解释“为什么升级有价值”，不打断主任务

## 5. 微信小程序 UI 要点

## 5.1 首页
- 顶部 Banner：1 个主入口 + 2 个快捷入口
- 功能宫格：8 宫格（转换、合并、压缩、拍照转 PDF 等）
- 最近使用：最近 10 条任务（轻量展示）
- 会员入口：权益摘要 + 明确 CTA

## 5.2 任务流程页
- 三步流：选功能 -> 选文件来源 -> 处理与分享
- 文件来源优先级：聊天文件、相册、拍照
- 完成态强化分享动作：微信好友/群聊

## 5.3 小程序视觉策略
- 更高按钮密度，更短路径
- 保持与 Web 同品牌色和图标语言
- 保留平台原生交互习惯（返回、弹层、授权）

---

## 6. 组件规范（跨端）

## 6.1 按钮
- 主按钮：`CTA` 绿底白字，禁用态 40% 不透明
- 次按钮：描边样式，提供明确 Hover/Active
- 所有可点击卡片与按钮使用 `cursor-pointer`（Web）

## 6.2 文件卡片
- 结构：图标 + 文件名 + 元信息 + 状态 + 行为
- 状态色：
  - 等待：中性色
  - 处理中：主色
  - 完成：成功色
  - 失败：错误色

## 6.3 进度条
- 行级进度（上传） + 任务级进度（合并/转换）
- 进度文案必须可读（百分比或阶段）
- 长时任务显示预计等待或排队提示

## 6.4 表单与提示
- 错误信息紧邻对应控件
- 错误容器需支持屏幕阅读（`aria-live` / `role="alert"`）
- 空状态必须告诉用户“下一步怎么做”

---

## 7. 响应式与可访问性

断点建议：
- `375`（小屏手机）
- `768`（平板）
- `1024`（小桌面）
- `1440`（大屏）

可访问性要求：
- 键盘可达：上传区、排序备用按钮、主 CTA
- 颜色不作为唯一状态表达
- 图片/图标提供可理解文本（含 `alt` 或隐藏说明）
- 焦点态明显可见

## 8. 关键页面线框建议（文本版）

### 8.1 Web 首页（简版）

3. 工具卡片网格（高频功能优先）

5. FAQ + Footer

### 8.2 Web 合并页（MVP）
1. 标题与说明
2. 拖拽上传区
3. 文件列表（可排序）
4. 校验与错误提示
5. `开始合并` 按钮 + 处理进度

### 8.3 小程序首页
1. 顶部价值 Banner
2. 常用功能宫格
3. 最近记录
4. 会员权益入口

## 10. 验收检查（设计侧）

- [ ] 无 emoji 作为功能图标，统一 SVG 图标集
- [ ] 所有交互元素具有可见 Hover/Focus（Web）
- [ ] 主任务链路 5 步内完成（进入 -> 上传 -> 设置 -> 处理 -> 下载）
- [ ] 错误均提供可执行恢复路径
- [ ] 亮色模式下文本对比度达标
- [ ] 关键页面在 375/768/1024/1440 无横向滚动
