## 2. 关键决策评审

### 2.1 核心处理服务：Rust vs Go

| 维度 | Rust (Actix-web) | Go (Gin) |
|------|------------------|----------|
| **性能** | ⭐⭐⭐ 极致性能，零成本抽象 | ⭐⭐⭐ 高性能，GC暂停 |
| **内存安全** | ⭐⭐⭐ 编译期保证，无GC | ⭐⭐ GC管理，内存泄漏风险 |
| **并发模型** | ⭐⭐⭐ Tokio异步，零成本 | ⭐⭐⭐ Goroutine，轻量 |
| **PDF生态** | ⭐⭐ pdfium, lopdf | ⭐⭐⭐ unidoc, pdfcpu |
| **团队熟悉度** | ⭐ 学习曲线陡峭 | ⭐⭐⭐ 国内普及度高 |
| **开发效率** | ⭐ 编译慢，调试难 | ⭐⭐⭐ 快速迭代 |
| **招聘** | ⭐ 人才稀缺 | ⭐⭐⭐ 容易招聘 |

**决策**: 使用 Rust
**理由**: 
- PDF处理是CPU密集型，Rust性能优势明显
- 内存安全避免处理大文件时的OOM风险
- 长期看技术债务更低

**风险**: 
- ⚠️ 团队学习成本高，需要培训时间
- ⚠️ 开发周期可能延长
- ⚠️ 第三方库生态不如Go成熟

**待决策**: 是否接受这些风险？是否需要Go作为备选方案？

---

### 2.2 数据库：MySQL vs PostgreSQL

| 维度 | MySQL 8.0 | PostgreSQL 15 |
|------|-----------|---------------|
| **JSON支持** | ⭐⭐ JSON函数 | ⭐⭐⭐ JSONB，索引 |
| **运维成熟度** | ⭐⭐⭐ 国内最广泛 | ⭐⭐ 相对少 |
| **复杂查询** | ⭐⭐ 够用 | ⭐⭐⭐ 更强大 |
| **云厂商支持** | ⭐⭐⭐ RDS完善 | ⭐⭐ 支持良好 |
| **团队熟悉度** | ⭐⭐⭐ 普遍熟悉 | ⭐⭐ 部分熟悉 |
| **性能** | ⭐⭐⭐ 读性能优秀 | ⭐⭐⭐ 写性能优秀 |

**决策**: 使用 MySQL 8.0
**理由**: 
- 团队运维经验丰富
- 云厂商RDS托管成熟
- MySQL 8.0 JSON功能已满足需求

**待决策**: 是否确认使用MySQL？是否需要评估PostgreSQL的JSONB优势？

---

### 2.3 消息队列：纯Redis vs 专业MQ

| 方案 | 组件 | 优点 | 缺点 |
|------|------|------|------|
| **方案A** | Redis Stream | 架构简单，运维少 | 功能有限，无持久化保证 |
| **方案B** | Redis + RabbitMQ | 功能丰富，可靠 | 多组件，复杂度高 |
| **方案C** | Redis + Kafka | 高吞吐，持久化 | 运维重，延迟高 |

**决策**: 方案A - 纯Redis Stream
**理由**: 
- PDF任务相对简单，不需要复杂路由
- 减少中间件依赖，降低运维成本
- 初期流量不大，Redis足够支撑

**风险**: 
- ⚠️ Redis重启可能丢消息（需配置AOF）
- ⚠️ 任务堆积时影响缓存性能
- ⚠️ 不支持延迟队列（需自行实现）

**待决策**: 是否接受Redis作为队列的限制？是否需要RabbitMQ作为备选？

### 2.4 部署架构：K8s vs Docker Compose

| 阶段 | 方案 | 适用场景 |
|------|------|----------|
| **MVP** | Docker Compose | 快速上线，单服务器 |
| **V2.0** | Kubernetes | 弹性伸缩，高可用 |
| **V3.0** | K8s + 私有化 | 企业版部署 |

**待决策**: 
- MVP阶段是否直接用K8s？
- 是否有私有化部署需求？
- 运维资源是否充足？

## 3. 技术风险评估

### 3.1 高风险项

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| Rust学习成本导致延期 | 高 | 中 | 预留2周学习时间，核心成员先学 |
| PDF转换质量不达标 | 高 | 中 | 多引擎备选，渐进优化 |
| 大文件处理OOM | 中 | 中 | Rust内存安全，流式处理 |
| Redis队列丢消息 | 中 | 低 | AOF持久化，监控告警 |

### 3.2 待解决问题

1. **Rust PDF库选择**
   - pdfium (Google Chrome内核) - 功能全，但依赖C库
   - lopdf (纯Rust) - 轻量，但功能有限
   - 是否需要绑定LibreOffice？

2. **文件存储策略**
   - 上传文件临时存储多久？
   - 转换结果文件保留多久？
   - 是否需要CDN加速下载？

3. **安全合规**
   - 用户文件是否加密存储？
   - 是否需要审计日志？
   - 等保合规要求？
