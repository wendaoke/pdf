## 核心用户故事

### US-001: PDF转Word
**作为** 一名大学生
**我想要** 将老师发的PDF课件转换成Word格式
**以便** 我可以添加自己的笔记和批注

**详细场景:**
小李收到老师发的PDF课件，想在重点内容旁边添加自己的理解和笔记，但PDF无法直接编辑。他需要将PDF转成Word，添加笔记后再打印出来复习。

**验收标准:**
- [ ] 支持上传最大100MB的PDF文件
- [ ] 转换后保留原始排版和格式（字体、段落、图片位置）
- [ ] 支持批量转换最多10个文件
- [ ] 转换时间不超过30秒/文件
- [ ] 转换成功率 > 95%
- [ ] 支持中文字符正确显示

**优先级:** P0

---

### US-002: Word转PDF
**作为** 一名应届毕业生
**我想要** 将Word简历转换成PDF格式
**以便** 投递给招聘公司时格式不会错乱

**详细场景:**
小张用Word写好了简历，但担心在不同电脑上打开格式会乱。他需要转成PDF确保格式一致，同时文件大小要适中方便邮件发送。

**验收标准:**
- [ ] 支持.doc和.docx格式
- [ ] 转换后格式与原Word完全一致
- [ ] 支持嵌入字体
- [ ] 转换时间 < 10秒
- [ ] 支持批量转换

**优先级:** P0

---

### US-003: PDF合并
**作为** 一名行政人员

**以便** 方便归档和发送

**详细场景:**
小王需要将本月的10份合同扫描件合并成一个PDF文件，按日期排序，方便存档和发送给领导审批。

**验收标准:**

### US-004: PDF拆分
**作为** 一名财务人员
**我想要** 将多页的PDF发票文件拆分成单页
**以便** 分别归档和报销

**详细场景:**

- [ ] 支持提取指定页面

- [ ] 拆分后文件命名规范（原文件名_页码）

### US-005: PDF压缩
**作为** 一名销售人员
**我想要** 压缩PDF文件大小
**以便** 能够通过邮件发送给客户

**详细场景:**
小张的产品手册PDF有15MB，超过了公司邮箱10MB的附件限制。他需要压缩到5MB以内，同时保持图片清晰度。

**验收标准:**
- [ ] 提供低/中/高三种压缩级别
  - 低压缩：文件减小10-20%，质量几乎无损
  - 中压缩：文件减小30-50%，质量良好
  - 高压缩：文件减小60-80%，质量可接受
- [ ] 预览压缩前后的文件大小
- [ ] 压缩后文字保持清晰可读
- [ ] 支持批量压缩
- [ ] 压缩时间 < 30秒/文件

**优先级:** P0

## 用户旅程地图

### 新用户首次使用流程

```
发现工具 → 访问网站 → 选择功能 → 上传文件 → 等待处理 → 下载结果 → 满意/不满意
```

**关键触点:**
1. **发现**: 通过搜索引擎/社交媒体/朋友推荐
2. **首页**: 3秒内理解产品价值
3. **功能选择**: 快速找到需要的功能
4. **上传**: 简单直观的上传体验
5. **处理**: 清晰的进度反馈
6. **结果**: 满意的输出质量
7. **留存**: 愿意再次使用或付费
