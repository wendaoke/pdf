## 4. 数据架构

### 4.1 数据库设计原则

- **主数据库**: PostgreSQL 存储业务数据
- **缓存层**: Redis 存储会话、热点数据

## 5. 安全架构

### 5.1 安全策略

| 层面 | 措施 |
|------|------|
| **传输安全** | 全站HTTPS，TLS 1.3 |
| **认证安全** | JWT Token + Refresh Token，RSA签名 |
| **接口安全** | 限流、防重放、参数校验 |
| **文件安全** | 病毒扫描、类型校验、大小限制 |
| **数据安全** | 敏感数据加密存储，数据库连接加密 |
| **隐私保护** | 文件自动清理，处理日志脱敏 |

### 5.2 文件安全流程

## 6. 性能设计

### 6.1 性能指标

| 指标 | 目标值 | 优化策略 |
|------|--------|----------|
| 首页加载 | < 2s | CDN + SSR + 资源压缩 |
| API响应 | < 200ms | 缓存 + 数据库优化 |
| 文件上传 | > 5MB/s | 分片上传 + 断点续传 |
| 转换处理 | < 30s/文件 | 异步队列 + 水平扩展 |
| 并发处理 | > 1000 QPS | 负载均衡 + 自动扩容 |

### 6.2 缓存策略

```
┌─────────────────────────────────────────────────────────┐
│                      多级缓存架构                         │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  L1: 浏览器缓存                                         │
│  ├── 静态资源: Cache-Control: max-age=31536000         │
│  └── 接口数据: ETag + Last-Modified                    │
│                                                         │
│  L2: CDN缓存                                            │
│  ├── 静态资源: 全球节点缓存                             │
│  └── 边缘计算: 就近响应                                 │
│                                                         │
│  L3: 应用缓存 (Redis)                                   │
│  ├── 会话数据: JWT黑名单                               │
│  ├── 热点数据: 用户信息、配置                          │
│  └── 限流计数: 滑动窗口计数器                          │
│                                                         │
│  L4: 数据库缓存                                         │
│  └── PostgreSQL: 共享缓冲区 + 查询计划缓存             │
│                                                         │
└─────────────────────────────────────────────────────────┘
```
