## 文档命名规范

### 文件命名规则
```
{类型}-{名称}-{版本}.md

示例:
- prd-pdf-converter-v1.0.md
- api-user-v1.0.md
- tc-pdf-merge-v1.0.md
```

### 版本号规则
- 格式: `v{主版本}.{次版本}.{修订}`
- 示例: `v1.0.0`, `v1.2.3`

## 文档模板

### PRD模板
```markdown
# {功能名称} PRD

## 1. 功能概述
## 2. 用户故事
## 3. 功能详细设计
## 4. 验收标准
## 5. 界面原型

## 7. 性能要求
```

```markdown
# {接口名称} API

## 接口地址
## 请求方法
## 请求参数
## 响应格式

```


## 文档更新流程

1. **创建**: 在对应目录下创建文档，遵循命名规范
2. **评审**: 提交PR或通知相关角色评审
3. **更新**: 修改时更新版本号，保留历史版本
4. **归档**: 废弃文档移至 `99-archive/history/`
